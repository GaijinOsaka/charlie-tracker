Design System

A warm, white, Claude-Code-inspired UI language for Charlie Tracker. Single source of truth for colors, type, spacing, and components — with a CSS variable patch ready to drop into src/App.css.

Tone: quiet, warm, precise. Pure white feels clinical for a daily-use dashboard; we use a paper-cream off-white instead. Single muted terracotta accent — applied sparingly so action items genuinely stand out. Serif headlines add personality without going corporate.

01

Color

A 4-tier surface system (page · card · soft · warn) plus a 3-tier ink system (primary · secondary · muted). One accent color, used only for primary actions and unread indicators. Status colors borrow the same desaturated, warm character.

07

Implementation order for Claude Code

Suggested order — each step is shippable on its own.

  1. Apply the token patch to App.css + add font links to index.html. The legacy aliases mean nothing breaks.
  2. Header: swap brand mark to a 32px terracotta square with "C", change h1 to --ct-font-display, drop subtitle font size to 13px / muted.
  3. Tab nav: replace bottom-border tabs with the pill tab bar (.tab-nav → segmented control inside a soft fill). Active tab gets a white surface + shadow-md.
  4. Filter bar: collapse the four labelled selects into one search input + 3 pill dropdowns. Remove the bottom border (the section heading below it does the divider job).
  5. Actions box: remove the orange left border + uppercase title; replace with a section heading ("Needs your attention") above a grouped-list card.
  6. Message card: kill the 4px left border on unread; add a 7px terracotta dot in the top row instead. Read state = soft surface + lighter font weight (not a different background tint).
  7. Source badges: consolidate .source-arbor + .source-gmail to a single neutral .ct-badge-source. Source is meta, not status.
  8. Buttons: primary (Mark done) gets terracotta fill; everything else neutral with line border. Drop the colored-border-only style.
  9. Other tabs (Events, Calendar, Documents, Settings): apply the same tokens; layout tweaks happen in a follow-up design pass.
  10. Cleanup: once everything migrates, remove the legacy aliases from the :root.
One rule for Claude Code: never hard-code a color, font-family, radius, or shadow. If a value is needed and there's no token for it, add a token first.
